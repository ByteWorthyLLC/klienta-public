# Klienta Sales and Access Flow

This document defines the standard paid-boilerplate fulfillment flow.

## 1. Discovery

- Buyer submits inquiry via https://byteworthy.io
- Confirm use case, plan tier, and entity details

## 2. Commercial Approval

- Issue quote or invoice
- Confirm payment and plan scope
- Confirm primary technical contact

## 3. License Issuance

- Issue commercial license token/record from private ops workflow
- Record customer ID, plan, seats, and expiration metadata

## 4. Repository Access

- Provision private GitHub access for authorized users only
- Apply least privilege (`pull`) by default
- Verify access acceptance

## 5. Onboarding

- Share setup docs and environment requirements
- Align on deployment target and security controls
- Confirm first successful deployment milestone

## 6. Ongoing Operations

- Track active seats and renewal dates
- Rotate/renew license material as needed
- Remove access promptly for offboarded users

## 7. Renewal / Upgrade

- Review active usage and seat needs
- Apply upgrade or renewal terms
- Re-issue license metadata where required
